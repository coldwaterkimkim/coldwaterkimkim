## How I Work

- 문제를 먼저 좁히고, 작게라도 실제로 만져볼 수 있는 흐름을 만듭니다.
- 랜딩, 카피, 인터랙션, 데이터 흐름, 검증 지표를 한 덩어리로 봅니다.
- 완성형 서비스라고 과장하기보다, MVP와 프리토타입의 경계를 명확히 두려고 합니다.
- 혼자 만든 결과물도 사용자 반응과 지표 앞에 세우는 연습을 하고 있습니다.
